# Задача: мульти-агентный пайплайн генерации SEO-статей

## Идея
Автоматизировать написание статей для блога MONTAS через цепочку агентов:
1. **Агент-эксперт** — пишет черновик статьи по теме, хорошо разбираясь в предметной области (в будущем — разные "профили" под разные ниши).
2. **SEO-агент** — адаптирует черновик под SEO-требования (см. чеклист ниже).
3. **Фактчекер (опционально, второй этап)** — проверяет черновик на достоверность фактов.
4. **Агент-иллюстратор (опционально)** — сам пишет промт по содержанию статьи и генерирует обложку/картинки (GPT Image / Nana Banana).
5. Готовая статья публикуется через существующий API редактора.

Конечная цель — сайт, который автоматически и непрерывно генерирует статьи, поднимается в поиске и передаёт трафик/ссылки на сайты клиентов, офера или в форму заявки.

## Что уже есть в проекте (используем, не переизобретаем)
- Редактор статей: `admin/index.html`
  - Создание/сохранение статьи: `saveArticle()` (~строка 945) → `POST /api/posts` или `PUT /api/posts/:id` (~963-967)
  - CRUD статей идёт через `apiFetch()` (~строка 708), JWT-авторизация через `doLogin()` (~729)
  - Текущий SEO-чекер (базовый): `runSeoCheck()` (~строка 1195)
  - Интеграции для публикации/анонса: Telegram и Dev.to уже настроены в `admin/config.js` (`window.MONTAS_CONFIG`)
  - Публикация в Dev.to через API: `api/devto.js`
- Backend `/api/posts`, `/api/login`, `/api/cases` — same-origin (`API = ''` в `admin/index.html:704`), реализация бэкенда не в этом репозитории (искать отдельно, если нужно менять серверную логику).

## Расширенный SEO-чеклист (добавить в SEO-агента / runSeoCheck)
- Ключевое слово в первом абзаце
- Ключевое слово в meta-описании
- Ключевое слово хотя бы в одном H2
- Плотность ключевого слова (1–3%)
- Alt-тексты у картинок
- Внутренние ссылки (есть ли хоть одна)
- Внешние ссылки (есть ли хоть одна)
- Длина абзацев (предупреждать, если слишком длинные)
- Уникальность заголовка среди существующих статей
- Slug содержит ключевое слово
- Читаемость: среднее количество слов в предложении

## Важно
- `admin/config.js` содержит реальные токены (Telegram bot token, Dev.to token) — не выводить их в публичные логи/коммиты и не палить third-party при работе с этим файлом.
- В репозитории есть и другие разделы сайта (главная, cases, blog) — они не относятся к этой задаче, трогать не нужно.

## Текущая задача

Реализовать MVP AI SEO Content Pipeline для MONTAS.

Цель:
- минималистичная flat admin-панель;
- этапы: эксперт-автор → SEO-агент → фактчекер → генерация изображения → черновик/публикация в CMS;
- первый этап реализации: writer → SEO-проверка по чеклисту → сохранение черновика через существующий API.

Ограничения:
- не менять main, cases, blog и другие несвязанные разделы;
- не выводить в логи токены из admin/config.js;
- SEO-критерии брать из ~/Downloads/TZ_SEO_CMS_Montas.docx.
