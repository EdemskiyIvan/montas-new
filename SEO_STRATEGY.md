# SEO стратегия MONTAS

## Как работает SEO

Люди вводят конкретные узкие запросы → попадают на наши статьи → оказываются в воронке на нашем сайте → оставляют заявку.

Google/Яндекс ранжируют страницы по трём факторам:
1. **Контент** — отвечает ли статья на запрос лучше конкурентов
2. **Авторитет домена** — сколько других сайтов ссылаются на нас
3. **Технические факторы** — скорость, индексируемость, структура (уже настроено)


## Цель

10+ заявок в месяц с органического трафика.

Путь:
- 2000–5000 уникальных посетителей/мес с органики
- Конверсия 1–3% → 20–50 целевых действий → 10–20 заявок


## Временная шкала

| Период | Что происходит |
|---|---|
| 1–3 мес | Статьи индексируются, трафик 0–200/мес |
| 3–6 мес | Позиции растут, 500–2000/мес |
| 6–12 мес | 2000–8000/мес, первые заявки |
| 12+ мес | Устойчивый поток |


## Ключевые инсайты

**Качество важнее количества**
- 1 глубокая статья (2000+ слов) > 10 поверхностных (300 слов)
- Слабые статьи снижают доверие Google к домену целиком
- Оптимум: 3–5 статей в неделю, каждая 1500–3000 слов
- 15–20 статей/мес — разумный потолок без риска

**Низкочастотные запросы**
Не «маркетинг», а «как настроить таргет вконтакте для строительной компании».
Таких запросов тысячи, конкуренция низкая, конверсия высокая потому что человек уже знает чего хочет.

**CTA расстановка**
- Крючок в начале статьи
- CTA в середине ("нужна помощь с этим? →") — люди уходят на 40–60% прочтения
- CTA в конце
- Не только в конце — обязательно в середине

**Внешние ссылки**
Когда другие ресурсы ссылаются на нас — Google повышает авторитет домена.
Публикация на Хабре, VC.ru, Дзен, Medium, Dev.to даёт реальные ссылки.
Важно: везде ставить canonical на montas.ru чтобы оригинал был у нас, не на Хабре.

**Кластеризация**
Группировать статьи по темам (SMM, контекст, дизайн) и делать между ними перелинковку.
Это показывает Google что мы эксперты в теме, а не просто пишем обо всём подряд.

**Метрики и масштабирование**
Яндекс.Метрика + Google Search Console → смотреть:
- Какие статьи дают трафик → писать похожие
- Какие запросы на позиции 4–15 → дожимать эти статьи
- Какие страницы дают заявки → масштабировать формат


## Три типа контента

| Тип | Цель | Раздел |
|---|---|---|
| **Статья** | Трафик, осведомлённость, верх воронки | /blog/ |
| **SEO-кейс** | Трафик + конверсия, середина/низ воронки | /blog/ |
| **Кейс** | Убеждение для тех кто уже на сайте | /cases/ |

**Схема работы кейсов:**
```
ПОИСК
  ↓
SEO-статья-кейс в блоге
"SMM для застройщика: 50 заявок за месяц"
  ↓
CTA внутри статьи
"Смотреть полный кейс →"
  ↓
Красивый кейс на сайте (с цифрами, дизайн)
  ↓
"Обсудить проект"
  ↓
ЗАЯВКА
```

**Пример переформатирования кейса под SEO:**
```
❌ "Кейс: лендинг для Tomoru"         (никто не ищет)
✅ "Как увеличить конверсию лендинга в 2 раза — реальный кейс"
✅ "Продвижение B2B SaaS в России: кейс с результатами"
✅ "SMM для строительной компании: от 0 до 50 заявок в месяц"
```


## AI SEO Content Pipeline

### Агенты

**1. Ресёрчер**
- Получает: ключевое слово / тему
- Делает: пишет глубокую статью 2000+ слов со структурой, подзаголовками, примерами
- Закрывает смежные вопросы по теме

**2. SEO-агент**
- Проверяет плотность ключевого слова (1–3%)
- Ключевое слово в первом абзаце, в meta-описании, в H1 и хотя бы одном H2
- Slug содержит ключевое слово
- Длина абзацев (не стены текста)
- Внутренние ссылки на другие статьи сайта
- Внешние ссылки (хотя бы одна)
- Читаемость: среднее количество слов в предложении
+ лёгкий фактчек ("найди фактические ошибки и исправь")

**3. Иллюстратор**
- Генерит обложку по заголовку и теме статьи
- GPT Image / аналог

**4. Публикация на montas.ru**
- Статья сохраняется в CMS как черновик
- Человек апрувит одной кнопкой
- При публикации генерируется статический HTML (уже реализовано)
- Sitemap обновляется автоматически (уже реализовано)
- Пинг Google Indexing API для быстрой индексации

**5. Интегратор**
Адаптирует и публикует на медиа-площадки через API:
```
Dev.to      — полная статья с canonical, API уже подключён
Habr        — полная статья с canonical
Дзен        — адаптированная версия с картинками (важно для Яндекса)
VC.ru       — выжимка 500 слов + "читать полностью →"
Medium      — полная статья с canonical
LinkedIn    — адаптация под формат
Telegram    — анонс + ссылка, бот уже подключён
```
Везде canonical указывает на montas.ru — ссылка идёт в плюс, дублирования нет.

**6. Аналитик** (отдельный цикл, раз в неделю)
- Смотрит Google Search Console + GA4
- Находит статьи на позициях 4–15 (чуть-чуть не в топ-3) → даёт задачу дожать
- Находит темы которые дают конверсии → масштабирует
- Находит статьи с высоким bounce rate → улучшает


## Технический фундамент (уже реализовано)

- Чистые URL: `montas.ru/blog/slug`, `montas.ru/cases/slug`
- Статический HTML генерируется при публикации (контент виден Google сразу)
- Sitemap.xml обновляется автоматически
- SSL, gzip, быстрая отдача через nginx
- Auto-deploy: git push → сайт обновляется за 5 сек
- canonical тег в каждой статье
- og:title, og:description, og:image для соцсетей
